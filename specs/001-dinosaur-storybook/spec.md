# Feature Specification: 恐龍故事書系統

**Feature Branch**: `001-dinosaur-storybook`  
**Created**: 2025-11-30  
**Status**: Draft  
**Input**: User description: "故事書系統創建 - 顯示恐龍介紹的兒童友善網站，包含圖片、搜尋、多語言支援和可愛的小故事"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 瀏覽恐龍介紹 (Priority: P1)

身為一位家長或孩童，我想要在首頁點擊「恐龍介紹」按鈕，進入恐龍介紹頁面，瀏覽各種恐龍的圖片與簡短介紹，讓孩子能夠輕鬆學習恐龍知識。

**Why this priority**: 這是網站的核心功能，沒有恐龍介紹內容，其他功能都沒有意義。使用者必須能看到恐龍資訊才有價值。

**Independent Test**: 可以獨立測試 - 點擊首頁的「恐龍介紹」按鈕後，應顯示恐龍圖片與介紹文字（名稱、生活時期、食性、發現地點），每個介紹控制在 200 字以內。

**Acceptance Scenarios**:

1. **Given** 使用者在首頁，**When** 點擊「恐龍介紹」按鈕，**Then** 進入恐龍介紹頁面，顯示第一隻恐龍的圖片和介紹資訊
2. **Given** 使用者在恐龍介紹頁面，**When** 頁面載入完成，**Then** 可以看到恐龍的名稱、生活時期、食性和發現地點
3. **Given** 恐龍介紹內容，**When** 顯示任一恐龍介紹，**Then** 文字篇幅不超過 200 字

---

### User Story 2 - 換頁瀏覽多隻恐龍 (Priority: P1)

身為一位孩童，我想要使用明顯的「上一頁」和「下一頁」按鈕來瀏覽不同的恐龍，讓我可以輕鬆看到更多恐龍介紹。

**Why this priority**: 換頁是瀏覽多筆恐龍資料的基礎功能，沒有換頁就只能看一隻恐龍，無法完整體驗網站內容。

**Independent Test**: 可以獨立測試 - 在恐龍介紹頁面，點擊「下一頁」按鈕應顯示下一隻恐龍，點擊「上一頁」按鈕應顯示上一隻恐龍。

**Acceptance Scenarios**:

1. **Given** 使用者在第一隻恐龍的介紹頁面，**When** 點擊「下一頁」按鈕，**Then** 顯示第二隻恐龍的介紹
2. **Given** 使用者在第二隻恐龍的介紹頁面，**When** 點擊「上一頁」按鈕，**Then** 顯示第一隻恐龍的介紹
3. **Given** 使用者在第一隻恐龍的介紹頁面，**When** 查看換頁按鈕，**Then** 「上一頁」按鈕不可用或隱藏
4. **Given** 使用者在最後一隻恐龍的介紹頁面，**When** 查看換頁按鈕，**Then** 「下一頁」按鈕不可用或隱藏
5. **Given** 換頁按鈕，**When** 顯示在頁面上，**Then** 按鈕圖案大且明顯，適合孩童點擊

---

### User Story 3 - 回到首頁 (Priority: P1)

身為一位使用者，我想要在任何恐龍介紹頁面都能看到「回首頁」按鈕，讓我可以隨時返回網站首頁。

**Why this priority**: 導航功能是基本需求，使用者必須能夠輕鬆返回首頁，否則會困在恐龍介紹頁面。

**Independent Test**: 可以獨立測試 - 在任何恐龍介紹頁面點擊「回首頁」按鈕，應返回網站首頁。

**Acceptance Scenarios**:

1. **Given** 使用者在恐龍介紹頁面，**When** 點擊「回首頁」按鈕，**Then** 返回網站首頁
2. **Given** 恐龍介紹頁面，**When** 頁面載入完成，**Then** 「回首頁」按鈕清楚可見

---

### User Story 4 - 點擊圖片查看大圖 (Priority: P2)

身為一位孩童，我想要點擊恐龍圖片來查看更大的圖片，讓我可以更清楚地看到恐龍的樣子。

**Why this priority**: 查看大圖增強使用體驗，但不是核心功能，沒有此功能使用者仍可閱讀恐龍介紹。

**Independent Test**: 可以獨立測試 - 點擊恐龍圖片後，應以較大尺寸顯示該圖片，並可關閉返回原頁面。

**Acceptance Scenarios**:

1. **Given** 使用者在恐龍介紹頁面，**When** 點擊恐龍圖片，**Then** 顯示該圖片的大圖檢視
2. **Given** 使用者正在查看大圖，**When** 點擊關閉或圖片外區域，**Then** 關閉大圖，返回原頁面

---

### User Story 5 - 搜尋恐龍 (Priority: P2)

身為一位使用者，我想要使用搜尋框來尋找特定種類的恐龍，讓我可以快速找到想看的恐龍介紹。

**Why this priority**: 搜尋功能提升便利性，但當恐龍數量不多時，換頁瀏覽即可滿足需求。

**Independent Test**: 可以獨立測試 - 在搜尋框輸入恐龍名稱或關鍵字，應顯示符合條件的恐龍列表。

**Acceptance Scenarios**:

1. **Given** 使用者在恐龍介紹區域，**When** 在搜尋框輸入「暴龍」，**Then** 即時過濾並顯示名稱或介紹中包含「暴龍」的恐龍
2. **Given** 使用者正在輸入搜尋關鍵字，**When** 每輸入一個字元，**Then** 結果列表即時更新（不需點擊搜尋按鈕）
3. **Given** 使用者搜尋關鍵字，**When** 沒有符合的結果，**Then** 顯示友善的「找不到結果」提示訊息
4. **Given** 搜尋結果列表，**When** 點擊某隻恐龍，**Then** 進入該恐龍的介紹頁面

---

### User Story 6 - 閱讀恐龍小故事 (Priority: P2)

身為一位孩童，我想要在恐龍介紹的最後閱讀一個相關的可愛小故事，搭配插圖，讓學習更有趣。

**Why this priority**: 小故事增加趣味性和教育價值，但屬於附加內容，沒有故事仍可瀏覽基本介紹。

**Independent Test**: 可以獨立測試 - 在恐龍介紹頁面向下捲動或換頁後，應可看到該恐龍相關的小故事和可愛插圖。

**Acceptance Scenarios**:

1. **Given** 使用者在恐龍介紹頁面，**When** 瀏覽至介紹末段，**Then** 顯示與該恐龍相關的小故事
2. **Given** 恐龍小故事區塊，**When** 故事顯示時，**Then** 穿插可愛的插圖

---

### User Story 7 - 切換語言 (Priority: P3)

身為一位使用者，我想要切換網站的顯示語言（中文或英文），讓不同語言背景的孩童都能閱讀。

**Why this priority**: 多語言是加值功能，初期可以先支援單一語言，後續再擴展。

**Independent Test**: 可以獨立測試 - 點擊語言切換選項後，頁面所有文字內容應切換為所選語言。

**Acceptance Scenarios**:

1. **Given** 使用者在任何頁面，**When** 選擇切換至英文，**Then** 頁面內容以英文顯示
2. **Given** 使用者在任何頁面，**When** 選擇切換至中文，**Then** 頁面內容以中文顯示
3. **Given** 使用者已選擇語言，**When** 瀏覽其他頁面，**Then** 語言設定保持不變
4. **Given** 使用者已選擇語言並關閉瀏覽器，**When** 再次造訪網站，**Then** 語言設定仍為上次選擇的語言（透過 localStorage 保存）

---

### Edge Cases

- 當恐龍資料檔案遺失或損壞時，系統應顯示友善的錯誤訊息
- 當恐龍圖片載入失敗時，應顯示預設的佔位圖片
- 當搜尋框輸入特殊字元時，系統應能正常處理不會崩潰
- 當只有一隻恐龍時，換頁按鈕應適當隱藏或停用

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系統 MUST 在首頁顯示「恐龍介紹」按鈕，點擊後進入恐龍介紹頁面
- **FR-002**: 系統 MUST 顯示恐龍的圖片、名稱、生活時期、食性和發現地點
- **FR-003**: 系統 MUST 將每個恐龍介紹控制在 200 字以內
- **FR-004**: 系統 MUST 提供明顯的「上一頁」和「下一頁」按鈕供換頁使用
- **FR-005**: 系統 MUST 提供「回首頁」按鈕讓使用者返回首頁
- **FR-006**: 系統 MUST 支援點擊恐龍圖片查看大圖
- **FR-007**: 系統 MUST 提供搜尋框讓使用者搜尋不同種類的恐龍，採用即時搜尋模式（邊打字邊過濾結果）
- **FR-008**: 系統 MUST 在恐龍介紹最後顯示相關的小故事和插圖
- **FR-009**: 系統 MUST 支援中文和英文兩種語言切換，預設語言為中文
- **FR-010**: 系統 MUST 使用瀏覽器 localStorage 儲存使用者的語言偏好設定
- **FR-011**: 系統 MUST 使用 JSON 檔案儲存恐龍資料（練習用途，不使用資料庫）
- **FR-012**: 系統 MUST 僅針對電腦瀏覽器進行設計，不需考慮手機版

### 設計要求

- **DR-001**: 介面風格 MUST 簡單可愛，類似兒童繪本或「動物森友會」風格
- **DR-002**: 換頁按鈕 MUST 大且明顯，方便孩童點擊
- **DR-003**: 單一頁面篇幅 MUST 適合孩童閱讀，不宜過長

### Key Entities

- **恐龍 (Dinosaur)**: 網站的核心內容單位，包含名稱、圖片、生活時期、食性、發現地點、介紹文字、相關小故事
- **小故事 (Story)**: 與恐龍相關的短篇故事，包含故事文字和配圖
- **語言設定 (Language)**: 使用者選擇的顯示語言（中文或英文）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 使用者能在 3 秒內從首頁進入恐龍介紹頁面並看到第一隻恐龍
- **SC-002**: 使用者能在 1 秒內完成換頁動作（點擊上/下一頁按鈕）
- **SC-003**: 5-10 歲兒童能獨立操作網站完成瀏覽 3 隻恐龍介紹的任務
- **SC-004**: 使用者能在 5 秒內找到並使用搜尋功能
- **SC-005**: 語言切換後，所有頁面內容在 2 秒內完成更新
- **SC-006**: 100% 的恐龍介紹內容符合 200 字以內的限制
- **SC-007**: 網站在主流桌面瀏覽器（Chrome、Firefox、Safari、Edge）上正常運作

## 恐龍資料內容需求 *(mandatory)*

開發網站時，需要同時建立以下恐龍資料和圖片素材：

### 初始恐龍清單（建議 5-8 隻）

| 恐龍名稱 | 生活時期 | 食性 | 發現地點 | 特色 |
|---------|---------|------|---------|------|
| 暴龍 (Tyrannosaurus Rex) | 白堊紀晚期 | 肉食性 | 北美洲 | 恐龍之王，擁有強大的咬合力 |
| 三角龍 (Triceratops) | 白堊紀晚期 | 草食性 | 北美洲 | 三隻角和大型頭盾 |
| 劍龍 (Stegosaurus) | 侏羅紀晚期 | 草食性 | 北美洲 | 背上有骨板，尾巴有尖刺 |
| 腕龍 (Brachiosaurus) | 侏羅紀晚期 | 草食性 | 北美洲、非洲 | 脖子超長，可以吃到高處的樹葉 |
| 迅猛龍 (Velociraptor) | 白堊紀晚期 | 肉食性 | 蒙古 | 聰明又敏捷的小型掠食者 |
| 翼龍 (Pteranodon) | 白堊紀晚期 | 肉食性（魚類） | 北美洲 | 會飛的爬蟲類，不是恐龍但常一起介紹 |
| 甲龍 (Ankylosaurus) | 白堊紀晚期 | 草食性 | 北美洲 | 全身裝甲，尾巴像槌子 |
| 副櫛龍 (Parasaurolophus) | 白堊紀晚期 | 草食性 | 北美洲 | 頭上有長長的冠，可以發出聲音 |

### 每隻恐龍需要的資料

1. **基本資訊**
   - 中文名稱 + 英文名稱
   - 生活時期
   - 食性（肉食/草食/雜食）
   - 發現地點
   - 體型描述（長度、高度、體重的簡單描述）

2. **介紹文字**（中英文各一份，每份 200 字以內）
   - 適合 5-10 歲兒童閱讀
   - 語氣活潑有趣
   - 使用簡單易懂的詞彙

3. **小故事**（中英文各一份）
   - 100-150 字的可愛小故事
   - 故事主角為該恐龍
   - 內容溫馨有趣，適合兒童

4. **圖片素材**
   - 主要圖片：恐龍的可愛卡通插圖（用於介紹頁面）
   - 故事插圖：1-2 張配合小故事的插圖
   - 圖片風格：可愛、卡通、兒童繪本風格
   - 圖片格式：PNG 或 WebP，建議尺寸 800x600 以上

### 資料檔案結構


## Clarifications

### Session 2025-11-30

- Q: 圖片素材來源策略？ → A: AI 繪圖工具產生（如 DALL-E、Midjourney）
- Q: 網站預設語言？ → A: 中文為預設語言
- Q: 語言設定持久化方式？ → A: 使用瀏覽器 localStorage 儲存
- Q: 搜尋功能觸發方式？ → A: 即時搜尋（邊打字邊過濾結果）
- Q: 恐龍瀏覽的初始顯示順序？ → A: 按 JSON 檔案中定義的順序顯示

## Assumptions

- 恐龍資料和圖片將在開發過程中一同建立，存放在 JSON 檔案和靜態資源中
- 初期恐龍數量約 5-8 隻，足以展示換頁和搜尋功能
- 恐龍瀏覽順序依照 JSON 檔案中定義的順序顯示，開發者可在 JSON 中精心安排展示順序
- 網站為靜態內容展示，不需要使用者登入或個人化功能
- 圖片素材將使用 AI 繪圖工具（如 DALL-E、Midjourney）產生，確保風格一致且符合「可愛卡通兒童繪本風格」
- 多語言內容將在 JSON 資料中預先翻譯好（中文和英文）
- 恐龍介紹內容將由開發者撰寫，確保適合兒童閱讀
